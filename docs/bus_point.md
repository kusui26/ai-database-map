# バス停（国土数値情報 P11 バス停留所 ＋ P36 高速バス停留所）

国土数値情報の **バス停留所 (P11)** と **高速バス停留所 (P36)** を合わせ、AI Database Map で駅×半径の **バス停数**（交通アクセス性の代理指標）を生成するための **データ理解＋集約設計ドキュメント**。本書は実データ（`data/バス停留所データ/`・`data/高速バス停留所データ/`）を全数読込・分析した結果と、それに基づく集約設計を記録する。実装は設計確定後（人口 `population_mesh.md`／地価 `land_price.md` と同様）。CRS の扱いは [`CRS.md`](./CRS.md) を参照。

---

## 1. 概要

| 項目 | バス停留所 (P11) | 高速バス停留所 (P36) |
|---|---|---|
| 出典 | 国土数値情報 **P11**（2022年度＝令和4年度版）| 国土数値情報 **P36**（2023年度＝令和5年度版）|
| 対象 | **民間路線バス・公営路線バス・コミュニティバス** | **高速バス**（都市間・長距離輸送）|
| 除外 | **高速バス・デマンドバスを除外**（2022年度版で分離）| 一般路線・コミュニティ・デマンドを除外 |
| 点数（全国）| **278,515点** | **9,224点** |
| 幾何 | 点（Point）| 点（Point）|
| CRS | **JGD2011（EPSG:6668）** | **JGD2011（EPSG:6668）** |
| ライセンス | オープンデータ CC_BY_4.0 | オープンデータ CC_BY_4.0 |
| 作成時点 | 概ね2022年8月 | 概ね2023年11月 |

- **P11とP36は対象バス種別が排他**（P11は一般路線・コミュニティ／P36は高速）。2022年度版でP11から高速バスが明示的に除外されたため、**同じバス系統が両データに重複することはない**。
- ただし**同一地点（駅前など）に一般バス停と高速バス停が併設**されることはある（§4で実測）。
- **合算してバス停数とする**のが目的。生の合算 287,739点には**事業者単位の重複**が多く含まれるため、**「物理的な停留所」へ重複排除**するのが最適（§4）。
- **過去（2010年度版＝平成22年度）P11 との増減率**も算出する（§8）。2010版は**属性レイアウトが現行と逆**・対象範囲(デマンド込)・座標系(JGD2000)・**ライセンス(非商用)**が異なるため、§8で全数分析の上、範囲合わせ（デマンド除外・≤25m重複排除）で正規化して `bus_gr` を算出。**実測：2010→現行で全国 −1.1%**（都市増・地方減）。⚠️**非商用ライセンスは要確認**。

> バス停は人口メッシュ（面積按分）や地価（点の中央値）と違い、**「半径内の点を数える」** だけの単純な集計。ただし**重複排除の定義**が品質の鍵。

## 4. 重複の実態 → 「バス停数」の定義（**最重要**）

「同一地点で複数のバス事業者の停留所がある場合は別々に整備」（公式仕様）のため、**生データには事業者単位の重複が多い**。全国実測:

### 4.1 同一座標の重複（複数事業者が同一地点）
| データ | 点数 | ユニーク座標 | 同一座標の重複 |
|---|---|---|---|
| P11 | 278,515 | 242,893 | **35,622（12.8%）** |
| P36 | 9,224 | 5,145 | **4,079（44.2%）** ← 高速バスは事業者重複が極端 |

### 4.2 P11 と P36 の共起（駅前などの併設）
- P36点のうち **10m以内にP11点あり：58.3%**／50m以内：70.8%／100m以内：77.3%。
- P36点が **50m以内に同名のP11点を持つ：49.4%**（＝高速バス停の約半数は一般バス停と同一地点）。

### 4.3 最近傍距離の分布 → 重複排除の閾値は「曖昧さなし」
合算287,739点の最近傍距離（自分以外）:
- **≤1m：26.2%**（＝同一座標の重複）、≤25m：28.4%、≤50m：29.4%、**中央値：201m**。
- → **「~0mに重複の塊」→「1〜50mはほぼ空」→「100〜500mに別停留所」** の明確な二峰。**重複排除の閾値は 1〜25m のどこでも同結果**（生245,304〜242,083）。

### 4.4 採用する定義：**「物理的な停留所」へ重複排除**
| 定義 | 全国数 | 評価 |
|---|---|---|
| 生の合算（P11+P36 全エントリ）| 287,739 | 事業者単位。**P36 44%・全体16%が重複**＝駅前/高速ハブで過大計上 → ✗ |
| **重複排除（≤25mで連結＝1停留所）** | **241,708** | **物理的な停留所数**。アクセス性の指標として妥当 → ✅ **採用** |

- **理由**：バス停を5社が共用しても「歩いて行ける停留所」は1つ。事業者重複（特にP36の44%）をそのまま数えると、**高速バスハブや多事業者停留所のある駅で数が水増し**され、アクセス性の信号が歪む。**物理停留所数**が最適な「バス停数」。
- **重複排除法**：合算点を **≤25mで連結成分クラスタリング**（または ~10–25m グリッドスナップ。二峰が明確なので閾値非依存）→ 各クラスタ＝1停留所（代表点＝重心）。
- 旧実装 [`open_data.ipynb`](../script/open_data.ipynb) は **単純concat（重複排除なし）** だった。本設計は重複排除を加えて精度を上げる。

## 7. 駅×半径 バス停数 集約ロジック（詳細設計）

### 7.0 確定仕様
| 項目 | 内容 |
|---|---|
| データ | **現行**: P11(2022 一般)＋P36(2023 高速)／**過去**: P11(2010・§8) |
| 集計 | P11+P36を合算 → **≤25m連結で重複排除（物理停留所）** → 駅半径R内の停留所数 |
| 半径 | **500m / 1km / 2km / 5km / 10km / 20km**（人口・地価と統一）|
| 出力 | `bus_n_{半径}`（総数）＋内訳 `bus_n_local_{半径}`（一般）/ `bus_n_hw_{半径}`（高速）＋増減率 `bus_gr_{半径}`（2010→現行・§8）|
| 欠損 | カウントは常に整数（**0＝バス停なし＝有意**, NaNにしない）。増減率は2010が0でNaN |
| 信頼性フラグ | **不要**（カウントは常に定義され不安定さがない）|
| CRS | 6668→Albers→`cKDTree`（2010は4612→6668・差<1m）|

### 7.1 集約パラダイムと座標処理
- **点カウント**：半径内の停留所点を数えるだけ（人口の面積按分・地価の中央値とは異なる単純集計）。半径方向は **単調増加**。
- P11（入れ子zip 47県）＋P36（サブフォルダ 47県）を読み、幾何＋ソース種別（一般/高速）を取得 → 6668→Albers投影 → 合算。

### 7.2 重複排除（物理停留所）
- 合算点を **≤25mで連結成分クラスタリング**（`connected_components` on `cKDTree.query_pairs(25)`）→ 各クラスタ＝1停留所、代表点＝重心。各クラスタに **一般(P11)/高速(P36) 含有フラグ**を付与（内訳用）。
- 二峰が明確（§4.3）なので **10〜25m のどの閾値でも結果ほぼ同一**（堅牢）。
- 全国実測：生287,739 → **重複排除 241,708 停留所**（一般含む 240,226 ／ 高速含む 4,694 ／ 併設 3,212）。

### 7.3 採用半径（密度実測）
重複排除後の停留所で、駅半径内カウントを実測:
| 半径 | 中央 | 平均 | p90 | n=0駅 | （参考）高速含む中央 |
|---|---|---|---|---|---|
| 500m | 3 | 3.9 | 9 | 16% | 0 |
| 1km | 10 | 12.4 | 27 | 10% | 0 |
| 2km | 31 | 41.3 | 96 | 6% | 0 |
| 5km | 140 | 205.6 | 477 | 2% | 2 |
| 10km | 448 | 676.2 | 1,564 | 1% | 7 |
| 20km | 1,295 | 2,127.1 | 5,425 | 0% | 20 |

→ **人口・地価と同じ 500m/1km/2km/5km/10km/20km を採用**（結合の一貫性）。小半径（500m–2km）が徒歩アクセスとして最も意味を持ち、大半径は都市バス密度を表す。高速バス停は小半径ではほぼ0でハブ駅に集中（内訳 `bus_n_hw` が拾う）。

### 7.4 出力カラム（`station_dataset` 本体に追記・`bus_` 接頭辞）
| 系統 | 列 | 型 | 内容 |
|---|---|---|---|
| 総数 | `bus_n_{500m,1km,2km,5km,10km,20km}` | int | 半径R内の**物理停留所数**（P11+P36・重複排除後）|
| 内訳:一般 | `bus_n_local_{R}` | int | 半径R内で**一般バス(P11)を含む停留所数** |
| 内訳:高速 | `bus_n_hw_{R}` | int | 半径R内で**高速バス(P36)を含む停留所数** |
| 増減率 | `bus_gr_{R}` | float(%) | 2010→現行 の停留所数 増減率（§8）|
| （任意）| `bus_near_dist_m` | int | 最寄バス停までの距離(m) |

- 内訳は **併設停留所（一般＋高速）を両方に数える**ため `local + hw ≥ 総数`（差＝併設。全国3,212）。`bus_n_hw` は「高速バスが使える停留所数」＝ハブ性の指標。
- **NaN方針**：総数・内訳は常に整数（0＝なし）。増減率は2010が0でNaN。
- 人口 `pop_*`・地価 `lp_*` と命名思想を統一（接頭辞 `bus_`）。

### 7.5 実装手順（擬似コード）
1. P11(2022・外zip→各県inner zip展開→geojson)・P36(2023・`/vsizip/`geojson)を全47県読み、幾何＋ソース種別を取得。
2. 6668→Albers投影、合算。
3. **≤25m連結で重複排除** → 停留所代表点（重心）＋ 一般/高速 含有フラグ。
4. 停留所で `cKDTree`、各駅 `query_ball_point(R)` → `bus_n` / `bus_n_local` / `bus_n_hw`。
5. 増減率は §8（2010を同手法で集計して比）。
6. `station_dataset.csv` に `grp` で右結合。検証：都心駅は多・地方駅は少、半径単調性 `bus_n_500m ≤ … ≤ bus_n_20km`。

### 7.6 設計決定ログ（経緯）
| 論点 | 検討 | 確定 | 根拠（実測）|
|---|---|---|---|
| 合算 | P11のみ / P11+P36 | **P11+P36** | 一般＋高速で網羅。種別は排他で系統重複なし |
| 数え方 | 生concat / 物理停留所 | **物理停留所（重複排除）** | P36 44%・全体16%が事業者重複＝生は過大 |
| 重複閾値 | 1m〜50m | **≤25m連結** | 最近傍が ~0m と 100m+ の二峰＝閾値非依存 |
| 半径 | バス局所 / 統一6半径 | **500m/1km/2km/5km/10km/20km** | 人口・地価と統一（結合一貫性）|
| 内訳 | 要否 | **一般/高速を併設両カウント** | `bus_n_hw`=高速ハブ性の指標 |
| 欠損 | 0→NaN? | **0は採用（有意）** | カウントは常に定義 |
| フラグ | 要否 | **不要** | 中央値のような不安定性がない |

## 8. 過去データ（2010年度版）と バス停数の増減率

「過去からの増減率」のため、**バス停留所 P11 2010年度（平成22年度）版**（`data/バス停留所データ/2010/`・47県個別zip・**254,842点**）を起点に、現行（2022/2023）との物理停留所数比を取る。**実データを全数分析した結果、2010版は属性レイアウト・範囲・座標系が大きく異なる**ため、慎重な正規化が必須（以下すべて実測）。

### 8.1 2010年度版 P11 の実態（全数分析）
| 観点 | **2010 P11（平成22年度・実測）** | 現行（P11 2022 ＋ P36 2023）|
|---|---|---|
| 点数 | **254,842点**（47県個別zip）| P11 278,515 ＋ P36 9,224 |
| **属性レイアウトが逆** | `P11_001`=名／**`P11_002`=区分コード**（'1,1,2,3'）／**`P11_003_*`=事業者名**／`P11_004_*`=系統 | `P11_001`=名／`P11_002`=事業者／`P11_003_*`=系統／`P11_004_*`=区分 |
| 多事業者 | **1点に複数事業者をカンマ統合＝既に物理停留所単位**（同一座標重複わずか2.7%）| 事業者別に分離（重複12.8%）|
| 区分コード | **{1,2,3,4,5}**（**4＝デマンド**=停留所の1.8%／**高速コード6は無し**＝高速は1/2に混在）| P11{1,2,3,5}＝デマンド・高速除外済／P36{6}＝高速 |
| 高速バス | **2010 P11 に内包**（独立コードなし→1/2に混在）| P36(2023)として独立（コード6）|
| CRS | **なし(.prj無) → JGD2000(EPSG:4612)** | JGD2011(EPSG:6668)|
| 文字コード・形式 | **Shift_JIS(cp932)・GML+shp**（geojsonなし）| UTF-8・geojson有 |
| ライセンス | **非商用** | CC-BY 4.0 |

> **データ整形ロジックへの影響（重要）**：2010は**列の意味が2022と逆**（区分コードが `P11_002`）。読込コードを**版で分岐**し、2010は **cp932・CRS=4612付与・`P11_002` から区分コード取得** が必要。

### 8.2 増減率の正規化アプローチ（実測で確定）
1. **多事業者の扱い差は重複排除が正規化**：2010(統合済・重複2.7%)も現行(分離・重複12.8%)も、**≤25m連結で物理停留所**に揃える → **同じ基準で比較可能**（最大の鍵）。
2. **デマンドを2010から除外**：区分コードが `4`/`0` のみの停留所を除く（＝**3,398点・1.3%**）。現行(デマンド除外済)と範囲一致。
3. **高速**：2010は1/2に混在・現行はP36。**両年とも高速を含む**ので総数比較は整合（ただし高速だけの増減率は2010で分離不可＝§8.4）。
4. **CRS**：2010を 4612→Albers、現行を 6668→Albers（差<1m）。
5. **増減率**：`bus_gr_{R} = (現行の半径内停留所数) / (2010の半径内停留所数) − 1`（カウント比・%）。2010が0の半径はNaN。新設・廃止＝アクセス変化そのものが信号なので **no-match（カウント比）で妥当**。

### 8.4 出力・注意・ステータス
- 出力：**`bus_gr_{500m,1km,2km,5km,10km,20km}`（%, 2010→現行・総数の増減率）**。2010が0の半径はNaN。
